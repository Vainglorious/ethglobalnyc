WHAT WE CAN SEE
---------------
default_v3.polymarket_markets_all  (== _active; 2,843,273 rows)
  The big Polymarket market catalog — ONE ROW PER OUTCOME. Cols: event_id,
  market_id, question, groupitemtitle, umaenddate, active, closed, token_id,
  outcome, outcome_price, condition_id, volume, insert_time.
  - 579,405 distinct events; 1,421,589 distinct condition_ids.
  - insert_time is a single batch snapshot (2026-06-13 20:07) — it's a current
    snapshot, NOT a time-series.
  - KEY: contains the GRANULAR match props that Gamma's "event" view hides —
    O/U totals, corners, exact scores, halves. This is how we find token_ids for
    markets PolyGun/Gamma-event don't surface.

umalabs.market_snapshots  (2,240 rows)  *** ODDS TIME-SERIES ***
  polymarket_id, captured_at (DateTime64 ms UTC), price_yes, price_no, volume,
  liquidity. 2,208 distinct markets, window 2026-06-10 17:19 -> 2026-06-13 18:44.
  This is the time-stamped odds history -> the raw material for "beat the market"
  and the timestamp gate. (Coverage is small/recent so far — see gaps below.)

umalabs.markets_current  (2,208 rows)
  Enriched current view per market + UMA resolution status: question, slug,
  event_slug, condition_id, question_id, closed, resolved_flag, uma_status_raw,
  end_time, closed_time, volume, liquidity, price_yes/no, odds_source, phase,
  confidence, has_uma_match, first_dispute_at, ingested_at.
  - Right now: all phase="proposed", resolved_flag=0, has_uma_match=0.

umalabs.uma_oo_v2_events_decoded  (2,387 rows)  *** UMA RESOLUTION ORACLE ***
  Decoded UMA Optimistic Oracle V2 chain events: block_timestamp, event_name,
  contract_address, topic1/2/3, data, tx_hash.
  - event_name: Settle 1177, ProposePrice 1177, RequestPrice 21, DisputePrice 12.
  - window 2026-05-29 -> 2026-06-12. This is the resolution/dispute feed for the
    Bet-3 "privileged info" arbitrage (know the proposed/settled outcome early).
  uma_oo_v2_events_raw (2,387)  -> raw chain logs behind the decoded view.
  uma_oo_v2_events_decoded_mv   -> materialized view (raw -> decoded).
  uma_oo_v2_event_signatures (4) -> topic0 -> event_name map (Settle/Propose/
                                    Dispute/Request).
  polymarket_markets_all_dispute (0) -> empty staging (markets_all shape + dispute_state).

HOW THIS FEEDS THE COLONY
-------------------------
- polymarket_markets_all  -> resolve any market/outcome to token_id + condition_id
  + current price (the "what can we bet + at what odds" lookup).
- market_snapshots        -> odds over time -> beat-the-market signal + the
  timestamp-gate corpus (query rows with captured_at <= as_of_ts).
- uma_oo_v2_events_decoded -> early resolution/dispute knowledge -> Bet-3 arb edge.
- umalabs is OUR sandbox (full DDL) -> we can build derived tables / the /query
  backend / the gate here without touching default_v3.
