KEY ARCHITECTURE FINDING — this env uses V3 (MPC / WaaS) wallets
  - The deprecated server endpoint /environments/{env}/embeddedWallets creates V2/Turnkey
    wallets and is REJECTED here: "Cannot create a Turnkey embedded wallet in an environment
    configured for V3 wallets."
  - The working server-side endpoint for V3 pregenerated wallets:
      POST {base}/environments/{ENV_ID}/waas/create
      headers: Authorization: Bearer <DYNAMIC_API_KEY>, Content-Type: application/json
      body:    { "identifier": "<email>", "type": "email", "chains": ["EVM"] }
      -> 201, returns user.verifiedCredentials[].address (eip155 EVM address),
         wallet_properties.version "V3", thresholdSignatureScheme "TWO_OF_TWO".
  - V3 = MPC: key shares held by Dynamic (2-of-2). **No raw private key is returned.**
    Signing happens THROUGH Dynamic (API/SDK), not via a local private key. This is the big
    contrast with our cast-generated treasury wallet (which we DO hold the key for).

WHAT THIS MEANS FOR COLONY WALLET ARCHITECTURE
  - Dynamic can pregenerate many ant wallets fast, server-side, with one API key. Good for
    "tens of ants" and beyond. Each wallet is tied to an identifier that a real user could
    later "claim" by logging in with that identifier.
  - Tradeoff vs. local cast keys:
      * Dynamic MPC: managed, no key to leak, claimable by humans, but SIGNING requires a
        Dynamic API/SDK call (not a local signer). Need to wire that for ant Arc txs.
      * cast local keys (treasury-style): we hold the key, can sign locally/instantly, but we
        manage secrets ourselves and there's no human-claim story.
  - Open question: how to SIGN an Arc tx from a Dynamic V3 wallet server-side (the WaaS signing
    API). Needed before ants funded via Dynamic can actually stake/transfer on Arc.

✅ SERVER-SIDE SIGNING WORKS + FULL CROSS-PRODUCT ROUTE (2026-06-13)
  Two server-side signing paths in @dynamic-labs-wallet/node-evm:
    1. DELEGATED (delegatedSignTransaction/Message/TypedData): needs walletId + walletApiKey +
       keyShare from a wallet.delegation.created flow. For wallets a USER delegates to us.
    2. DEVELOPER-CREATED (DynamicEvmWalletClient): we create the wallet and HOLD the key share,
       so we can sign immediately — no delegation needed. THIS is what we used.
  Recipe (developer-created, server-side):
    const c = new DynamicEvmWalletClient({ environmentId });
    await c.authenticateApiToken(apiKey);                       // exchanges API key -> JWT
    const w = await c.createWalletAccount({ thresholdSignatureScheme:"TWO_OF_TWO", password, backUpToDynamic:true });
    const walletClient = await c.getWalletClient({ walletMetadata:w.walletMetadata, password,
        externalServerKeyShares:w.externalServerKeyShares, chainId:5042002, rpcUrl });
    // walletClient is a viem WalletClient that signs via MPC -> use for Arc txs AND Unlink evm.fromViem.
  Proven: created an MPC wallet in ~2.2s and signed a message; then signed real Arc txs.

  COMBINED A->B->C->D ROUTE (dynamic/combined-route.mjs) — ALL HOPS PROCESSED ✅
    A treasury 0x1be3...2D8F --3 USDC native (Arc)-->        B Dynamic MPC 0xDF53...E984
    B --depositWithApproval 2 USDC (B signs via MPC)-->      C Unlink (private)  -> C private 3 USDC
    C --Unlink private transfer 1 USDC-->                    D Unlink (private)
    Result: deposit processed, transfer processed, C private 2 USDC, D private 2 USDC.
  Key point: the SAME Dynamic MPC walletClient both sends Arc USDC AND acts as Unlink's EVM
  provider (evm.fromViem) for the deposit. Dynamic + Unlink compose cleanly.

  ARC NATIVE/ERC-20 NOTE: native USDC (18dp) and the 0x3600 ERC-20 (6dp) are the SAME balance.
  Funding a wallet with native USDC also gives it spendable ERC-20 USDC + gas. Verified on-chain.

WALLET-MODEL TAKEAWAY FOR COLONY
  Hybrid is viable and proven: treasury (local key) funds Dynamic MPC wallets (managed, no key to
  leak, claimable by humans -> ties to Worldcoin verified lineages), which can push USDC into
  Unlink's private layer. Decide per-ant: Dynamic MPC for verified/human roots, local cast keys
  for cheap ephemeral ants.
