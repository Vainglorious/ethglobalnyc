THE LESSON (the important one)
- PolyGun's Telegram path is TOO SLOW and TOO FRAGILE for last-minute execution.
  It choked exactly when timing mattered and a human had to step in. This is the
  concrete failure case that justifies building the fast self-custody path.
- NEXT: build the Node executor on @polymarket/clob-client (sub-second, no Telegram,
  carries the current order version the Python client lacks). See
  notes/2026-06-21-clob-blocking-and-trade-automation.txt for why Python CLOB is
  schema-blocked (NOT geoblocked — that's proven cleared from Toronto).
- Operationally: NEVER run >1 pg.py at once (shared pg.session lock). One bet, let
  it finish, verify on-chain, then the next.
