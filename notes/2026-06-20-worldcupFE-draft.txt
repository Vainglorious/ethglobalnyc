WHAT THIS IS
------------
A temporary, World-Cup-themed surface added to the existing WorldColony frontend
(the vanilla Three.js "dinasty" app served by Vite). It tells the betting story:
the AI ant colony's real on-chain trades, simulated/counterfactual trades, the
match schedule with colony picks, and the strategy (arbitrage / UMA / privileged
ClickHouse data). Built to be ripped out cleanly after the tournament.

Full design + decisions live in: notes/2026-06-20-worldcupFE.md
This file is the short build/handoff summary.


ARCHITECTURE (important)
------------------------
- NOT a separate page/route. It's an IN-APP OVERLAY inside index.html.
- A prominent entry button (top-left) opens a full-screen layer (#wc-overlay) that
  covers the whole 3D app + HUD with a top z-index. A "Back to the Colony" button
  takes the entry button's spot to dismiss it. Esc also closes.
- All content renders (vanilla JS, innerHTML) into #wc-content.
- Everything is fenced with "WORLD CUP TEMP — remove after the tournament" comments
  so teardown = delete the fenced blocks + the 3 worldcup/ files.

DATA SOURCING
-------------
- Fixtures use the SAME source as the backend: colony/data/world_cup_kg.json (built from
  OpenFootball 2026), extracted to public/data/worldcup-games.json (teams/date/time/
  group/venue/score/played). Logic mirrors colony_api _forecast_games_from_kg.
- Trades read from the two copied JSON snapshots. NOTE: these are snapshots — re-copy
  from repo root if the source files change.


HOW TO RUN
----------
  cd frontend && npm install
  npm run dev -- --port 3000      # http://localhost:3000  (vite default is 5173; we override)
Click the top-left "World Cup Predictions" button to open the overlay.


DEPLOY (Vercel)
---------------
Nothing special: it's all static under public/ and ships into dist/ via `vite build`.
No routes/rewrites needed (overlay, not a page). Adil is handling the Vercel deploy.
