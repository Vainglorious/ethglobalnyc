TESTNET vs MAINNET — the answer to "does it matter / how does owning mainnet affect testnet"
  - Mainnet ENS and Sepolia ENS are SEPARATE systems (different chains, registries).
    They do NOT affect each other. Owning worldcolony.eth (mainnet) has ZERO effect
    on colonny.eth (Sepolia), and vice versa — they're different names anyway.
  - The agent subnames (colonny.eth) being on TESTNET is fine for the colony's
    internal identity demo, BUT a judge resolving a name in a normal/mainnet ENS
    tool will only see MAINNET names. So worldcolony.eth (mainnet) is the one that
    matters for the live "resolve the name, see the colony" showcase.

STEPS — make worldcolony.eth showcase the trades
-------------------------------------------------
A) FORWARD record (this is the showcase: name -> trades wallet). You CAN do this:
   1. app.ens.domains/worldcolony.eth, connect as owner 0x9e91D1E6…dfa9a4.
   2. If no resolver set: set Resolver = ENS Public Resolver (1 tx, mainnet gas).
   3. Records -> Addresses -> set ETH address = 0xe9E32Ca24aa1eF725F650b5489281FE621363AA9
      (a valid EVM address; judges then view it on POLYGONSCAN for the trades).
   4. Optional text records to tell the story:
        url = link to predictions.json / the dashboard
        description = "Colony agent trading wallet — live Polymarket bets"
        avatar = colony image
   => Now resolving worldcolony.eth returns the trades wallet. THIS is the demo move.

B) PRIMARY name (reverse: address -> worldcolony.eth) — IMPORTANT LIMITATION:
   A primary/reverse record must be set BY THE ADDRESS ITSELF (needs its private key).
   - We do NOT control 0xe9E3 (PolyGun custodial) -> we CANNOT make worldcolony.eth
     its primary name. So 0xe9E3 won't *display* "worldcolony.eth" on explorers.
   - We CAN set worldcolony.eth as the primary name of a wallet WE control
     (0x9e91D1E6… main, or 0xcc16… treasury) — but those don't hold the trades.
   => For showcasing trades, use the FORWARD record (A). Primary-name on the trades
      wallet is not possible given the custodial setup.
