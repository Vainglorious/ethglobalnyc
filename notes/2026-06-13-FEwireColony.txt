1) colony_api/  — NEW FastAPI backend wrapping the colony harness
----------------------------------------------------------------
Runs the existing harness as a managed subprocess; exposes runs, artifacts, and
an SSE stream to the frontend. Source of truth stays the CLI harness.
Deployed on RAILWAY:  https://ethglobalnyc-production.up.railway.app

Key endpoints:
  GET  /health                      service check
  GET  /config                      backend contract + defaults + identity_fields
  POST /runs/demo                   start a 200-agent LLM run
  GET  /runs/{id}                   poll status + artifact links
  GET  /runs/{id}/stream            SSE: status + colony_event (+ KG events)
  GET  /runs/{id}/agents            every agent_record incl. ENS + wallet + latest_forecast{side,stake}
  GET  /runs/{id}/rooms             debate rooms (participants, stance, claims, synthesis)
  GET  /ants                        200 ants from committed public wallet store (no run needed)
  GET  /kg/world-cup [/summary]     committed tournament KG (204 entities / 588 rels)
  POST /scouting/run                public-data KG scouting (default: Brazil vs Morocco, DeepSeek scouts)
  GET  /runs/{id}/kg [/manifest] [/scouting-audit]   generated KG artifacts

Defaults (from /config): agents=200, rooms=12, seed=205, voice_mode=llm,
  agent_wallets=true, wallet_provider=dynamic,
  wallet_store=colony/data/agent-wallets.dynamic.200.public.json
LLM via OpenRouter/DeepSeek (deepseek-v4-flash) — Railway env vars.

---------------------------------------------

ENV COLONY_API_RUNS_DIR=/data/runs, EXPOSE 8000.

3) Frontend went LIVE against the API
-------------------------------------
New: frontend/public/dinasty/  (databridge.js, ants.js, kgview.js, hud.js, colony.js, main.js)
Flow (per colony_api/README "Frontend Interaction"):
  - databridge.js loads latest successful run from GET /runs
  - "Get ants" -> GET /ants -> binds wallet/ENS identity to visible ants
  - "Get KG" -> GET /kg/world-cup -> renders static tournament KG
  - "Run scouting" -> POST /scouting/run -> SSE stream renders real kg_entity/kg_relationship events
  - "Run LLM agents" -> POST /runs/demo -> seeds colony stats + thought ticker
  - Fallback to /data/demo.jsonl if backend unavailable
  Backend URL config: frontend/public/config.js (window.DN_CONFIG.API_URL)
SSE event types: status, colony_event; scouting adds kg_stage/kg_entity/
  kg_relationship/kg_manifest/scouting_audit (REAL run output, not fake animation).
