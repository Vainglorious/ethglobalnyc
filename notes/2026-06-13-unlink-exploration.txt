GOAL
  Understand the Unlink protocol and see ONE simple ~1 USDC private transfer between two

WHAT UNLINK IS
  "Build private applications on blockchains." A smart contract on an existing EVM chain
  (no separate chain, no bridge). You deposit ERC-20 tokens in, then transfer them PRIVATELY
  between Unlink accounts (no on-chain balances/amounts/history), then withdraw back out.
  Docs: https://docs.unlink.xyz  (clean .md pages + https://docs.unlink.xyz/llms.txt)
  Saved reference: unlink/docs/unlink-overview.md

  Four operations: depositWithApproval() in, transfer() private, withdraw() out,
  execute() to call contracts (needs a seed-backed account).

WHY IT MATTERS FOR COLONY
  Unlink supports arc-testnet (chainId 5042002) — the SAME Arc chain Colony settles money on
  (USDC = native gas there). So Unlink is a candidate privacy layer for ant-to-ant USDC
  (private stakes/inheritance). For this first "does it work" test we lean base-sepolia
  (simplest, best documented).

SDK + ACCOUNTS (key facts)
  - Install: npm install @unlink-xyz/sdk@canary
  - Packages: /browser (MetaMask), /client (custodial Node — what we use), /admin (apiKey +
    registration + auth tokens), /crypto (account constructors, alt import path).
  - Account constructors: fromMnemonic, fromSeed, fromEthereumSignature, fromMetaMask, fromKeys.
    NO fromPrivateKey — so we use mnemonics. Need appId + chainId; wrong chainId = different account.
  - getAddress() returns an "unlink1..." address.
  - transfer({ recipientAddress, token, amount }) -> tx.wait() -> status "processed"|"failed".
    amount is in smallest unit as a string (NOT decimals-adjusted).
  - faucet.requestPrivateTokens({ token, amount }) funds the Unlink account (shielded);
    requestTestTokens() mints ERC-20 to an EVM wallet.

✅ DONE — 1 USDC A->B PRIVATE TRANSFER WORKS (2026-06-13)
  Resolved the token blocker: the Arc testnet USDC ERC-20 is 0x3600000000000000000000000000000000000000

  unlink/deposit-and-transfer.mjs (uses viem walletClient over Arc RPC + evm.fromViem; deposits
  via depositWithApproval, then transfer).

  Run result:
    Treasury 0x1be3...2D8F (20 USDC) --depositWithApproval 2 USDC--> A : status processed
    A private balance: 2000000 (2 USDC)
    transfer 1 USDC A->B : status processed

  Key SDK facts used: evm.fromViem({ walletClient, publicClient }) to sign on-chain from a raw
  key in Node; client.depositWithApproval({ token, amount, evm }); client.transfer({...}); both
  return handles -> .wait() -> status "processed". balanceOf(token) returns smallest-unit string
  (null when zero).
